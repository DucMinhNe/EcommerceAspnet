## Overview

EcommerceAspnet is the server-side API for an e-commerce platform. It exposes a clean set of
REST endpoints for managing the full storefront domain — products, categories, providers, carts,
orders, customers, employees, comments, and payment methods — backed by a PostgreSQL database
through Entity Framework Core.

The API ships with JWT bearer authentication, an interactive Swagger/OpenAPI explorer, static
file hosting for uploaded images, permissive CORS for browser-based clients, and a Dockerfile +
Compose setup so it can run alongside a separate React admin front end.

## Features

- **Catalog management** — full CRUD for products, product categories, and providers, including
  product image upload to `wwwroot` and a soft-delete / restore flow (`PUT /api/Products/Restore/{id}`).
- **Cart & orders** — endpoints for carts, orders, and order details to model the complete
  purchase lifecycle.
- **Customer accounts** — customer registration and login, plus customer address management.
- **Staff / admin** — employee and job-title management for back-office operations.
- **Comments** — product comment endpoints with image attachments.
- **Payment methods** — manage the payment options available at checkout.
- **JWT authentication** — separate register/login flows for employees and customers
  (`/api/Auth/employee/...`, `/api/Auth/customer/...`) issuing signed JWT bearer tokens.
- **Swagger UI** — interactive API documentation generated by Swashbuckle (enabled in development).
- **CORS enabled** — open `AllowAnyOrigin` policy for integration with front-end SPAs.
- **Container-ready** — multi-stage `Dockerfile` and `docker-compose.yml`.

## Getting started

### Prerequisites

- [.NET 7 SDK](https://dotnet.microsoft.com/download/dotnet/7.0)
- A running [PostgreSQL](https://www.postgresql.org/) instance

### 1. Clone and restore

```bash
git clone https://github.com/DucMinhNe/EcommerceAspnet.git
cd EcommerceAspnet
dotnet restore
```

### 2. Configure the connection string and JWT

Update the `DefaultConnection` connection string and `JWT` settings in `appsettings.json`
to match your environment:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=ecommerce;Username=postgres;Password=yourpassword"
  },
  "JWT": {
    "ValidIssuer": "your-issuer",
    "ValidAudience": "your-audience",
    "Secret": "your-long-random-secret-key"
  }
}
```

> For real deployments, keep secrets out of source control — use User Secrets, environment
> variables, or a secrets manager rather than committing them to `appsettings.json`.

### 3. Set up the database

The repository includes a SQL schema dump (`ecommerce.sql`) you can load directly into PostgreSQL:

```bash
psql -U postgres -d ecommerce -f ecommerce.sql
```

The project is also EF Core enabled, so once migrations are added you can apply the schema with:

```bash
dotnet ef database update
```

### 4. Run

```bash
dotnet run
```

The API starts on `http://localhost:5256` (and `https://localhost:7283`). In development, browse to
`/swagger` to explore and try the endpoints.

### Run with Docker

```bash
docker compose up --build
```

The API is published on port `8080` and the companion React admin front end (if present in a
sibling `../ecomerce_admin_react` directory) on port `3000`.
